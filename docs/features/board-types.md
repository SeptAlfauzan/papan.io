# Board Types Feature

## Overview

Core TypeScript types shared across the infinite canvas application. Defines the data contracts for strokes, camera, history, connection status, and tools.

## Types

### Point
```ts
type Point = [number, number]
```
A 2D point in world coordinates. Stored as tuple `[x, y]` for performance — avoids object allocation overhead in hot loops (render, hit test).

### Stroke
```ts
interface Stroke {
  id: string
  points: Point[]
  color: string
  width: number
  minX: number  // bounding box
  maxX: number
  minY: number
  maxY: number
}
```
A single drawing stroke on the infinite canvas.
- `id`: Unique identifier (generated by `useCanvas.uid()`)
- `points`: Array of world-space coordinates
- `color`: Hex color string
- `width`: Stroke width in world-space units
- `minX/maxX/minY/maxY`: Axis-aligned bounding box for viewport culling optimization

### StickyNote
```ts
interface StickyNote {
  id: string
  x: number
  y: number
  width: number
  height: number
  text: string
  truncate: boolean
  color: string
}
```
A sticky note on the canvas.
- `id`: Unique identifier
- `x, y`: Top-left position in world coordinates
- `width, height`: Dimensions in world-space units
- `text`: Note content
- `truncate`: Whether to truncate overflow text
- `color`: Background hex color

### CameraState
```ts
interface CameraState {
  x: number
  y: number
  zoom: number
}
```
Camera/viewport state.
- `x, y`: World-space coordinate at screen center
- `zoom`: Scale factor (1 = 100%)

### HistoryEntry
```ts
interface HistoryEntry {
  type: 'add' | 'erase'
  strokes: Stroke[]
}
```
Command-pattern entry stored in undo/redo history.
- `type`: `'add'` = stroke was added, `'erase'` = stroke was removed
- `strokes`: Full Stroke objects (not just IDs) so undo/redo works correctly with out-of-order sync

### ConnectionStatus
```ts
type ConnectionStatus = 'connecting' | 'connected' | 'disconnected' | 'error'
```

### ToolMode
```ts
type ToolMode = 'pencil' | 'hand' | 'eraser' | 'sticky-note'
```

## Usage

```ts
import type { Stroke, Point, StickyNote, CameraState, HistoryEntry, ConnectionStatus, ToolMode } from '@/types/board.types'
```

## Design Decisions

### Tuple vs Object for Points
Points stored as `[number, number]` tuples rather than `{x, y}` objects. Reduces GC pressure in canvas render loops and simplifies array operations.

### Bounding Box on Stroke
Each stroke carries its AABB computed at draw time. Avoids expensive recomputation during viewport culling and hit testing. Updated incrementally as points are added during drawing.

### Full Stroke in History
History entries store complete Stroke objects rather than IDs. This ensures undo/redo remains consistent when strokes arrive out-of-order via WebSocket sync (e.g., a stroke erased locally before the add event arrives from server).
